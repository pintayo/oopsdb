# Contributing to OopsDB

First off, thank you for considering contributing to OopsDB! It's people like you that make it such a great tool for the community.

## How to Contribute

1. Check our [Issues](https://github.com/pintayo/oopsdb/issues) page to see if someone else has already created a ticket for your bug or feature request. If not, go ahead and make one!
2. Fork the repository and create a branch with a descriptive name.
3. Implement your fix or feature.
4. Test your changes locally to ensure nothing is broken.
5. Create a Pull Request with a clear title and description detailing what your changes do and why they're needed.

Feel free to ask for help on your PR. We're happy to guide you!
